Project Overview

Developed a playable Pong game on an 8x8 LED matrix, programmed using Arduino C++. The project demonstrated embedded game design, real-time user interaction, and efficient use of limited hardware resources.

Key Features

  • Single-player Pong gameplay with paddle cotnrolled via joystick.
  • 8x8 LED display to visually represent the ball and paddles.
  • Collision detection logic for paddle hits and wall bounces.
  • Adjustable ball speed and difficulty scaling for extended playability.

Technical Implementation

The game was programmed in C++ on an Arduino, using an 8x8 LED matrix to display the moving ball and paddle. A joystick was used to control paddle's movement. A main game loop handled ball movement, collision detection, and score tracking. Logic was added to increase ball speed over time, creating a dynamic challenge. The design required optimizing code to efficiently refresh the LED matrix while maintaining smooth gameplay.

Technologies Used:

  • Hardware: Arduino microcontroller, 8x8 LED matrix, joystick.
  • Software: Arduino IDE (C++) for game logic and display control.
  • Processes: Game loop design, collision detection, matrix refresh optimization.

Challenges & Solutions

  • Display limitations: The 8x8 matrix offered low resolution; overcame this by designing simplified paddle and ball representations.
  • Smooth animation: Early versions had flickering; solved by optimizing LED refresh rates.
  • Input responsiveness: Early paddle controls felt sluggish; improved by refining input polling and reducing latency.

Results

The final game successfully delivered a fully playable Pong experience on an 8x8 LED matrix, demonstrating skills in embedded programming, game logic design, and hardware interfacing under hardware constraints.
