Project Overview

Hospital delirium is a serious condition affecting patients — particularly elderly patients — who lose track of time, become disoriented, and miss routine activities like eating and exercise. This wearable device, built as part of ECE 198 at the University of Waterloo, combats delirium by delivering timely reminders to patients through an OLED display and audio speaker. The patient wears the device on their wrist, sets the time using a rotary dial, and receives prompts to eat and exercise at appropriate intervals — all without needing a phone or external device.

Hardware Components

Arduino UNO R4 Minima (Renesas RA4M1)

Main microcontroller handling all firmware logic, peripheral communication, and interrupt-driven state management.

ADXL335 3-Axis Accelerometer

Tracks patient movement and activity levels. Raw analog output is read, noise-suppressed, and compared against a threshold to detect motion.

SSD1306 128x32 OLED Display (I2C)

Displays current time and instructions to the patient. Communicates with the microcontroller over the I2C protocol.

Rotary Potentiometer

Used to set the current time by hour. Interrupt-driven input handling detects value changes and updates the internal clock arithmetic in real time.

Speaker + Amplifier Module

Delivers audio reminders to the patient — prompting them to eat or exercise at configured intervals.

3D-Printed Armband Enclosure

Custom-designed in Fusion 360. Houses all electrical components, with the OLED visible on the front and the speaker mounted on top. Secured with standoffs for easy access.

How It Works

▶ Device powers on, patient sets time via rotary potentiometer
↓ Internal clock begins tracking time using custom arithmetic firmware
↓ ADXL335 continuously reads 3-axis acceleration, noise suppressed via threshold
↓ If 30+ minutes elapsed → OLED + speaker prompt patient to eat
↓ If activity threshold not met → OLED + speaker prompt patient to exercise
↓ 1 potentiometer click → shows current time
↓ 3 potentiometer clicks → displays instructions
↺ Loop continues, monitoring activity and time

Firmware Implementation

All firmware was written in embedded C++ on the Arduino UNO R4. The core challenge was managing multiple peripherals simultaneously without an RTOS — achieved through interrupt-driven state management across the OLED, speaker, accelerometer, and potentiometer inputs.

The time-keeping system required custom clock arithmetic to handle hours, minutes, and seconds accurately while the potentiometer could adjust the hour at any point. Accelerometer noise was suppressed by sampling multiple readings and applying a configurable motion threshold before triggering any output.

Technologies Used:

  • Microcontroller: Arduino UNO R4 Minima (Renesas RA4M1)
  • Firmware: Embedded C++ — interrupt-driven state management, custom clock arithmetic, noise suppression
  • Communication Protocols: I2C (OLED display)
  • Sensors: ADXL335 3-axis accelerometer, rotary potentiometer
  • Output: SSD1306 OLED display, speaker + amplifier module
  • Enclosure: Fusion 360 CAD, 3D printing, standoff hardware
  • IDE: Arduino IDE

Challenges & Solutions

  • Multi-peripheral state management without RTOS: Coordinating the OLED, speaker, accelerometer, and potentiometer simultaneously required careful interrupt-driven design to avoid blocking delays stalling other inputs.
  • Accelerometer noise: Raw analog readings from the ADXL335 were noisy and triggered false motion detections. Solved by sampling multiple readings and applying a configurable threshold before registering activity.
  • Clock drift from potentiometer adjustment: Allowing the user to change the hour mid-session required careful arithmetic to avoid minutes and seconds miscalculating. Custom offset tracking variables were introduced to maintain accuracy across adjustments.
  • Enclosure fit: Fitting all components — board, display, speaker, and wiring — into a wrist-worn form factor required several Fusion 360 iterations and prototype prints to achieve a clean, wearable result.

Results

The final prototype successfully delivers time-based eating and exercise reminders through both visual (OLED) and audio (speaker) output, with reliable patient-facing time display and instruction prompts accessible via the rotary dial. The device operates fully standalone — no phone, app, or external connection required — making it suitable for a real hospital ward environment.
